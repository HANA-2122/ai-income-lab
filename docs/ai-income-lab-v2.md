# AI Income Lab v2 — 設計ドキュメント

## 概要

v2 は「貼るだけ評価」システムです。
X・記事・YouTube で見つけた AI 収益化ネタを `input.txt` に貼り付けると、
採用/保留/不採用を自動判定し、本質抽出・転用案を生成します。

## v1 との違い

| 項目 | v1（既存） | v2（新規） |
|-----|----------|----------|
| 入力形式 | 構造化Markdown（`## Case N:`） | 自由貼り付け（`---`区切り） |
| 評価軸 | 5項目100点スコア | 17項目・採用タイプ分類 |
| 出力 | reports/daily/*.md + results/*.csv | analysis.md + data/research_log.md |
| 用途 | 複数事例のバッチ処理 | 1件ずつ丁寧な評価・メモ化 |
| 依存 | analyze.js → report.js → idea.js | evaluate_idea.js 単独 |

## ファイル構成

```
input.txt               ← ここにネタを貼る（毎回上書き）
analysis.md             ← 最新評価レポート（毎回上書き）
data/
  categories.json       ← 7カテゴリ定義・キーワード辞書
  research_log.md       ← 累積ログ（追記モード）
scripts/
  evaluate_idea.js      ← メインスクリプト
docs/
  ai-income-lab-v2.md   ← このドキュメント
```

## 使い方

### 1. input.txt にネタを貼る

```
---
[タイトル（任意）]
https://x.com/...（URL自動検出）

ここに投稿文・記事の概要・自分のメモを自由に貼り付ける。
フォーマット不要。---で区切れば複数件OK。
---
```

### 2. 評価を実行する

```bash
npm run evaluate
```

### 3. 結果を確認する

```bash
# 最新の評価レポートを開く
code analysis.md

# 累積ログを確認する
code data/research_log.md
```

## 評価項目（17項目）

| 項目 | 説明 |
|-----|------|
| title | タイトル |
| source_url | 元URL（自動検出） |
| category | 7カテゴリのどれに当たるか |
| decision | 採用 / 保留 / 不採用 |
| adoption_type | 直接採用 / 構造採用 / 参考 / 不採用 |
| adoption_score | 0〜100点（4軸合計） |
| reason_for_decision | 判定理由 |
| core_mechanism | 本質的な仕組み |
| monetization_model | 収益モデル |
| evidence_strength | 高 / 中 / 低 |
| risk_level | 高 / 中 / 低 |
| red_flags | 検出された危険サイン |
| transfer_ideas | 転用案（3件） |
| recommended_next_action | 次のアクション |
| priority_rank | 今回の評価内でのランク |
| short_summary_for_x | Xへの投稿サマリー（140字） |
| claude_code_build_idea | Claude Codeで作れるもの |
| notes | 注意事項 |

## adoption_type の意味

| タイプ | 意味 | 例 |
|-------|------|-----|
| 直接採用 | そのまま小さく検証してよい | Polymarket観察ツール |
| 構造採用 | 直接実行は危険だが本質に転用価値 | AI美女 → 人格管理ツール化 |
| 参考 | 危険サインや市場観察として保存 | 中リスクで証拠弱い事例 |
| 不採用 | 新規性・転用価値が低い | 情報商材・証拠なし |

## 判定基準（10ルール）

1. 収益額をそのまま信じない
2. 本質が抽出できるかを見る
3. 実在する市場・ツール・制度に基づいているかを見る
4. Claude Codeで小さく検証できるかを見る
5. 法務・規約・BAN・倫理リスクを見る
6. 既存カテゴリと重複していないかを見る
7. 直接実行より、周辺ツール化・分析ツール化・メディア化できるかを見る
8. 短期の爆益主張より、再利用可能な構造を重視する
9. 「AIが魔法のように稼ぐ」ではなく、「どの工程をAIで圧縮しているか」を見る
10. 逆説的に、実行者より周辺ツール提供者の方が儲かる可能性も検討する

## 赤旗（red_flags）検出パターン

1. 一晩で爆益
2. Claudeに頼むだけ
3. 月額20ドルで大金
4. Telegram誘導
5. LINE誘導
6. 自動売買
7. 成人向けコンテンツ
8. 政治扇動リスク
9. 規約違反の可能性
10. 著作権/商標リスク
11. 証拠なしの収益主張

## スコアリング（4軸×25点=100点満点）

| 軸 | 配点 | 高評価条件 |
|----|------|-----------|
| 証拠強度 | 25点 | GitHub公開・スクリーンショット・決済証拠 |
| リスクの低さ | 25点 | 赤旗なし・規約準拠・自動売買なし |
| Claude Code実装可能性 | 25点 | 1〜2日でMVP化できるか |
| 市場妥当性 | 25点 | 既存カテゴリ一致・需要が実在する |
| 構造ボーナス | +5点 | 既知カテゴリで転用価値あり |

**判定閾値**: 採用≥60点 / 保留35〜59点 / 不採用<35点

## 7つの既存カテゴリ

1. AI架空人格・AIインフルエンサー系
2. Polymarket / 市場非効率検出系
3. UGCプラットフォーム量産系
4. Claude Code SNS自動化系
5. 市場・物件・商品スクリーニング系
6. AI音楽カタログ販売系
7. マーケティング / EC運用OS系
