# パターンマイニング プロンプト

Claude / ChatGPTに渡して使う。事例の「表面」ではなく「構造」を抽出するためのプロンプト集。

---

## 基本プロンプト（単一事例の深掘り）

```
以下のAI活用事例を分析してください。
事例をそのまま真似することが目的ではありません。
「この事例の裏にある構造」を抽出することが目的です。

[事例内容をここに貼る]

以下の観点で分析してください：

1. 本質的な能力
   - この事例が証明している「AIが可能にした能力」は何ですか？
   - 「月○万円稼いだ」という表面ではなく、「何ができるようになったか」を言語化してください。

2. なぜ人が反応したか
   - なぜ収益が出たか / なぜ拡散したか / なぜ注目されたか
   - 感情的な理由（不安・欲望・怠惰・好奇心）も含めて分析してください

3. 誰の悩みに使えるか
   - どんな職業・状況の人がこの能力を欲しがっているか
   - 月いくらなら払うと思うか

4. 5つの転用アイデア
   以下のジャンルを必ず含めてください：
   - 投資/MT5または XAUUSD への転用
   - Polymarket への転用
   - SEOサイト または ゲームルール解説サイト への転用
   - AI画像販売 への転用
   - Claude Code初心者向け教材 への転用

   各アイデアについて：
   - 商品の形（何を売るか）
   - 想定顧客
   - MVP（今週試せる最小版）
   - 最初の配布先
   - 収益化方法
   - 成功指標

5. 罠の分析
   - この事例のどこに罠があるか（規約・コスト・競合・陳腐化）
   - 真似したら失敗しやすいポイント

出力は日本語で、各項目をMarkdown形式で返してください。
```

## 複数事例の横断分析プロンプト

```
以下の複数のAI活用事例を横断して分析してください。

[cases.csvの内容またはレポートを貼る]

横断分析の観点：

1. 共通する能力パターン
   - 複数の事例で繰り返し登場している「AI能力」は何ですか？
   - それは汎用的か、特定ニッチに限られるか？

2. 共通する需要シグナル
   - 複数の事例で共通して「困っている人」の姿が見えますか？
   - その需要の大きさを推定してください

3. ギャップ分析
   - 見つかった能力に対して、まだ誰も手をつけていない転用先はどこですか？
   - 日本語圏で英語圏に比べてまだ少ない事例は何ですか？

4. 自分の強みとの接点
   以下の強みジャンルとの接点を見つけてください：
   - 投資 / MT5 / XAUUSD / Polymarket
   - SEO / ブログ
   - Claude Code 教材 / チュートリアル

5. 今すぐ試せる実験TOP3
   - 優先度が高い理由
   - 1日以内に試せる方法
   - 成功の判断基準
```

## シグナルタイプ別の見方ガイド

### revenue_claim（収益主張）を見るとき
- 売上か利益か → コスト・時間・API代を差し引いた実態は？
- 再現に必要な条件 → スキル・ツール・時間・資金
- 本当に再現可能か → 「自分が同じ状況で試したら？」を必ず考える

### capability_demo（能力デモ）を見るとき
- 「すごい」という感想で終わらせない
- 「これが可能なら、別の何に使えるか？」を5個考える
- デモの難易度 → 自分でも再現できるか？できるとしたら何日かかるか？

### pain_signal（需要シグナル）を見るとき
- 「誰か作ってほしい」「面倒くさい」→ それを自動化するとしたら何が必要か
- 支払い意思 → 「無料なら使う」か「お金を払ってでも欲しい」かを区別する
- 代替手段の有無 → 既存ツールで解決できるか？できないなら参入余地あり

### suspicious_claim（情報商材疑い）を見るとき
- 感情訴求の構造 → どの不安・欲望に訴えているか
- ファネル構造 → 無料コンテンツ→有料誘導のパスはどうなっているか
- 需要の実在性 → 「こういう悩みを持つ人は実在する」というシグナルとして活用

## 転用発想チェックリスト

事例を見たとき、必ず以下を5分で考える：

- [ ] 投資/MT5への転用：トレーダーのどの作業が自動化できるか
- [ ] Polymarketへの転用：予測市場参加者のどの情報ニーズに応えられるか
- [ ] SEOサイトへの転用：どのキーワードで記事を自動生成できるか
- [ ] AI画像販売への転用：どのニッチジャンルの画像需要があるか
- [ ] ゲーム/教育への転用：どのルールや概念を視覚化できるか
- [ ] Claude Code教材への転用：この事例を再現するチュートリアルを書けるか
