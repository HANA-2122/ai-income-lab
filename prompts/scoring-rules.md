# スコアリングルール（100点満点）

`scripts/score.js` の採点基準の詳細版。
手動評価や Claude Code での深掘り評価時に使用。

---

## 1. 需要（0〜20点）

市場規模・検索ボリューム・再現事例の多さを評価。

| スコア | 基準 |
|-------|------|
| 18〜20 | SEO/コンテンツ、SaaS/ツール：検索需要が高く競合事例多数 |
| 14〜17 | SNS自動化：需要はあるが規約リスクあり |
| 10〜13 | AI画像販売、Polymarket：ニッチだが成長中 |
| 5〜9  | その他：需要不明 |
| 0〜4  | 情報商材：市場はあるが参入価値が低い |

**確認方法**: Googleトレンド、Ahrefs/Ubersuggest で検索ボリューム確認

---

## 2. 収益化しやすさ（0〜20点）

主張されている収益額と収益化速度で評価。
**売上ではなく純利益（コスト差引後）を推定すること。**

| スコア | 収益規模（月次） |
|-------|----------------|
| 20    | 50万円以上（証拠あり） |
| 17    | 10〜50万円（証拠あり） |
| 14    | 5〜10万円（証拠あり） |
| 10    | 1〜5万円（証拠あり） |
| 6     | 1万円未満 or 証拠なし |
| 8     | 収益額不明（中程度と仮定） |

**注意**: API費用、広告費、ツール代、時間コストを差引くこと。

---

## 3. 再現性（0〜20点）

同じことを自分が試せるかを評価。

| スコア | 基準 |
|-------|------|
| 20 | GitHubにコード公開 + 詳細なREADME + 動作確認済み |
| 12〜15 | コード公開あり or 手順の説明あり |
| 5〜11 | 概要説明のみ、独自スキル不要 |
| 1〜4  | 特殊な人脈・コネ・独占的アクセスが必要 |

---

## 4. 自分の強みとの相性（0〜20点）

このプロジェクトでの「自分」= Claude Codeユーザー・エンジニア志望を前提。

| スコア | 基準 |
|-------|------|
| +8 | Claude Code / Claude API を活用 |
| +4 | Node.js / Python を使用 |
| +4 | n8n / Make / Zapier を使用 |
| +2 | その他の自動化ツール |
| 最低4 | 基準点 |

---

## 5. リスクの低さ（0〜20点）

法規制・規約違反・詐欺・技術的リスクを評価。

| スコア | 基準 |
|-------|------|
| 20 | 法的リスクなし、規約準拠、オープンソース |
| 11〜15 | 軽微なリスクあり（利用規約の確認が必要）|
| 5〜10 | 規約グレーゾーン or 価格変動リスク |
| 3〜4  | 規約違反の可能性 or Polymarket等の投機的要素 |
| 0〜2  | 情報商材 or 詐欺の可能性 or 自動売買 |

---

## 証拠強度ボーナス（+0〜5点）

| ボーナス | 基準 |
|---------|------|
| +5 | 強（複数の独立した証拠：GitHub + 売上スクリーンショット等） |
| +2 | 中（1つの証拠：スクリーンショットのみ等） |
| 0  | 弱・なし（主張のみ） |

---

## 最終判定

| スコア | アクション |
|-------|-----------|
| 75〜100 | **優先実験**: 今週中に小さな実験を開始 |
| 55〜74 | **詳細調査**: 元ソース・GitHub・コストを深掘り |
| 35〜54 | **様子見**: 1ヶ月後に再評価 |
| 0〜34  | **スキップ**: 時間を使わない |
| リスク高 | **スキップ（高リスク）**: リスク関係なくスコアに関わらず除外 |

---

## 煽り・詐欺チェックリスト

以下に3つ以上該当する場合、リスクを「高」に引き上げること:

- [ ] 「誰でも」「確実に」「絶対に」の多用
- [ ] 月収100万円以上の主張（個人・副業）
- [ ] 収益のスクリーンショットなし or 加工の可能性
- [ ] 匿名アカウント（フォロワー少・設立直後）
- [ ] 高額な講座・情報商材への誘導
- [ ] 「残り○席」「期間限定」の煽り
- [ ] 具体的な手法・コードを公開していない
- [ ] 否定的なコメントに攻撃的に反応
